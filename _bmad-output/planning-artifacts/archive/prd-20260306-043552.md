## Executive Summary

**Team Mode** transforms Atoms Plus from a single-agent AI coding assistant into a collaborative multi-agent development platform. Users describe their intent in natural language, and 8 specialized AI agents—Architect, Engineer, Product Manager, Data Analyst, Researcher, Project Manager, SEO Specialist, and Team Leader—collaborate in real-time to deliver comprehensive solutions.

Built on **LangGraph 1.0** for production-grade state management and orchestration, Team Mode provides:

- **Real-time collaboration visibility** via WebSocket streaming of agent thoughts, decisions, and actions
- **Multi-perspective problem solving** where each agent brings domain-specific expertise
- **Chinese LLM support** (Alibaba Qwen, DeepSeek, Zhipu GLM) via LiteLLM integration
- **Minimal-invasive integration** with the existing OpenHands codebase and Atoms Plus architecture

**Target Users:** Solo developers and small teams seeking the collaborative dynamics of a full engineering organization without the overhead.

**Problem Solved:** Current AI coding tools operate as single-persona assistants. Complex software development requires diverse perspectives—architecture, implementation, testing, documentation, UX—working together. Team Mode brings this multi-disciplinary collaboration to AI-assisted development.

### What Makes This Special

**Core Differentiator:** Unlike single-agent AI coding tools that provide batch responses from one perspective, Team Mode shows users the *collaboration happening*—watching the Architect design system boundaries while the Engineer considers implementation constraints, seeing the PM decompose requirements while the Analyst validates data assumptions.

**Key Insight:** The best software emerges from diverse expertise working together. Team Mode replicates this dynamic with specialized AI agents, each with distinct prompts, tools, and decision-making patterns orchestrated through LangGraph's state machine architecture.

**Why Now:** LangGraph 1.0's production stability (October 2025), mature Chinese LLM APIs, and proven WebSocket streaming patterns make this technically feasible. Market research confirms LangGraph as the optimal framework for 2026—34.5M monthly downloads, native streaming support, and minimal integration overhead.

## User Journeys

### Journey 1: Solo Developer - Success Path (Alex)

**Opening Scene:**
Alex is a solo developer working on a SaaS startup. It's 2 AM, and she's staring at her screen, overwhelmed by the scope of her next feature: a real-time analytics dashboard. She needs to design the data model, build the API, create the React components, and write comprehensive tests. Alone, this would take 2 weeks.

**Rising Action:**
She opens Atoms Plus and activates Team Mode. She types: "Build a real-time analytics dashboard with time-series charts, user segmentation, and export to CSV."

The Architect agent activates first, proposing a data architecture using WebSocket streams. The Engineer starts discussing implementation patterns. The PM breaks down the feature into discrete user stories. Alex watches the collaboration unfold in the right panel—each agent's thoughts streaming in real-time with colored role indicators.

**Climax:**
Fifteen minutes in, Alex sees something remarkable: the Architect and Engineer are debating whether to use server-sent events or WebSockets. The PM interjects with a user story about "real-time updates must feel instantaneous." The Engineer shifts approach based on this input. Alex realizes she's watching the kind of design discussion that used to require three senior engineers.

**Resolution:**
Within 2 hours, Alex has a working dashboard with comprehensive tests. The code is architected properly, documented by the agents, and follows best practices she wouldn't have known to apply. She pushes to production, gets positive user feedback, and returns to Team Mode for her next feature.

## Domain-Specific Requirements

### AI/ML Multi-Agent System Constraints

#### LLM Provider Considerations

| Constraint | Requirement |
|------------|-------------|
| **Rate Limiting** | Graceful degradation when any LLM provider hits rate limits; automatic fallback to alternative providers |
| **Token Budget** | Per-session token limits to prevent runaway costs; user-visible budget tracking |
| **Model Availability** | Health checks for all 17 supported models; real-time status indicators |
| **Chinese LLM Compliance** | Alibaba/DeepSeek/Zhipu terms of service compliance; content filtering awareness |

#### Agent Behavior Reproducibility

| Concern | Mitigation |
|---------|------------|
| **Non-Determinism** | Checkpoint state includes random seeds where possible; temperature logging |
| **Prompt Drift** | Version control for Microagent prompts; A/B testing infrastructure |
| **Cross-Model Consistency** | Behavioral baselines per role; regression tests for role behavior |

#### Real-Time Streaming Requirements

| Requirement | Specification |
|-------------|--------------|
| **WebSocket Stability** | Automatic reconnection with state recovery; heartbeat monitoring |
| **Back-Pressure Handling** | Client-side buffering for burst events; graceful degradation |
| **Multi-Client Support** | Same session viewable by multiple clients (team collaboration) |

### Integration Requirements

**LiteLLM Dependency:**
- All LLM calls routed through LiteLLM for provider abstraction
- Custom `ChatTongyi` wrapper for Alibaba-specific features
- Unified error handling across providers

**OpenHands Codebase Integration:**
- Maximum 5% of core files modified
- All new code isolated in `atoms_plus/team_mode/`
- No breaking changes to existing API contracts

**Microagent System:**
- 8 role Microagents in `.openhands/microagents/role-*.md`
- Trigger-based activation mechanism preserved
- No modifications to OpenHands Microagent loader

### Risk Mitigations

| Risk | Probability | Impact | Mitigation |
|------|-------------|--------|------------|
| **LLM Provider Outage** | Medium | High | Multi-provider fallback; cached responses for common patterns |
| **Agent Infinite Loop** | Low | High | Max iteration limits; timeout enforcement; user interrupt capability |
| **Token Cost Explosion** | Medium | Medium | Per-session budgets; admin alerts; model routing optimization |
| **WebSocket Disconnection** | High | Medium | Checkpoint recovery; automatic reconnection; offline queue |
| **Cross-Agent Deadlock** | Low | High | Timeout-based deadlock detection; priority-based resolution |

## Innovation & Novel Patterns

### Detected Innovation Areas

**1. Visible Multi-Agent Collaboration (Primary Innovation)**

Unlike black-box AI coding assistants, Team Mode makes agent collaboration *visible* in real-time. This challenges the assumption that "users don't need to see AI thinking—just give them results."

| Aspect | Innovation |
|--------|------------|
| **What's New** | Real-time visibility into agent discussions, disagreements, and consensus |
| **Novel Combination** | WebSocket streaming + LangGraph StateGraph + Role-based UI |
| **User Value** | Transforms AI assistance from "magic output" to "collaborative reasoning you can follow and trust" |

**2. Specialized Agent Roles with Distinct Personalities**

8 agents with different prompts, tools, and behavioral patterns—not just different names.

| Aspect | Innovation |
|--------|------------|
| **Assumption Challenged** | "One AI persona is sufficient for all coding tasks" |
| **Implementation** | Microagent prompt architecture + LangGraph node routing + Domain-specific tooling |

**3. Chinese + Western LLM Unification**

First developer tool to treat Chinese LLMs as first-class citizens with per-role model routing.

| Aspect | Innovation |
|--------|------------|
| **What's New** | Seamless switching between Qwen/DeepSeek/GLM and Claude/GPT/Gemini |
| **Market Position** | Unique value proposition for Asia-Pacific market |
