## 一、项目概述

Atoms Plus 是一个 AI 驱动的低代码开发平台，目标是复刻并超越 [atoms.dev](https://atoms.dev) 的核心功能。基于 OpenHands 开源框架进行深度定制和扩展，实现了多模型竞速、多角色 Agent 协调等差异化能力。

## 二、实现思路与关键取舍

### 2.1 技术选型

| 层级 | 技术栈 | 选型理由 |
|------|--------|----------|
| **前端** | React + TypeScript + TanStack Query | 类型安全、数据缓存管理成熟 |
| **后端** | Python + FastAPI + LiteLLM | 异步高性能、多 LLM 统一接口 |
| **数据库** | Supabase (PostgreSQL) | 开发效率高、实时订阅、免费额度 |
| **部署** | Vercel (前端) + Railway (后端) | 快速部署、自动 CI/CD |

### 2.2 架构设计决策

**决策 1：基于 OpenHands 而非从零开发**
- ✅ 复用成熟的 Agent 执行引擎、沙箱环境、事件系统
- ✅ 节省 80% 基础设施开发时间
- ⚠️ 代价：需要理解和适配复杂的现有架构

**决策 2：Race Mode 采用并行执行 + 流式聚合**
- ✅ 多模型同时请求，总耗时 = 最慢模型耗时（而非累加）
- ✅ 支持实时流式展示各模型响应
- ⚠️ 代价：需要处理并发控制、错误隔离

**决策 3：Agent Roles 使用 YAML 配置而非硬编码**
- ✅ 新增角色只需添加配置文件，无需改代码
- ✅ 支持运行时热更新
- ⚠️ 代价：需要实现配置加载和验证逻辑

**决策 4：Orchestrator 采用 DAG 任务调度模式**
- ✅ 支持并行/串行/依赖关系的复杂任务编排
- ✅ 自动聚合多角色输出
- ⚠️ 代价：前端 UI 复杂度较高，暂未完成

### 2.3 关键取舍

| 取舍项 | 选择 | 原因 |
|--------|------|------|
| 支付系统 | 暂不实现 Stripe | 优先完成核心 AI 功能 |
| 一键部署 | 暂不实现 | 需要云基础设施投入 |
| Orchestrator 前端 | 仅完成后端 | 后端 API 已可用，前端 UI 复杂 |
| 多租户 | 简化为单用户 | Demo 阶段优先跑通核心流程 |

## 五、项目亮点

### 5.1 工程亮点

- **统一多模型接口**：通过 LiteLLM 封装，一套代码支持 15+ 模型
- **可扩展角色系统**：YAML 配置驱动，新增角色零代码
- **完整事件溯源**：所有 Agent 动作可追溯、可回放

### 5.2 用户体验亮点

- **流式响应**：实时显示 AI 输出，无需等待完整响应
- **多模型对比**：Race Mode 让用户直观比较不同模型能力
- **一站式开发**：终端 + 浏览器 + 编辑器 + Git，无需切换工具

### 5.3 创新点

- **Orchestrator 多角色协调**：业界首创，将复杂任务自动分解给专业角色
- **国产模型深度支持**：阿里通义、DeepSeek、智谱 GLM 等一键接入

## 六、本地运行

```bash
# 克隆仓库
git clone https://github.com/Alenryuichi/atoms-plus.git
cd atoms-plus

# 安装依赖
make build

# 配置环境变量
cp .env.example .env
# 编辑 .env 填入 LLM API Key

# 启动
make run
```
